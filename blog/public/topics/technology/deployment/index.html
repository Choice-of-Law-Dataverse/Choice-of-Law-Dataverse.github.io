Multi Container Deployment on Kubernetes#

Kubernetes is an open-source container orchestration platform. It can be used to automate the deployment, scaling, and management of containerized applications.

Key Concepts#

  1. Pods

    • The smallest deployable unit in Kubernetes.
    • A Pod can host multiple containers that share networking and storage resources.

  2. Nodes

    • Physical or virtual machines that run Pods.
    • Components:
      • kubelet: Ensures containers are running in Pods.
      • kube-proxy: Manages networking and load balancing.
      • Container Runtime: Software responsible for running containers (e.g., Docker, containerd).

  3. Control Plane

    • The management layer that makes global decisions (e.g., scheduling).
    • Components:
      • API Server: Exposes the Kubernetes API.
      • Controller Manager: Ensures cluster state matches desired state.
      • Scheduler: Assigns Pods to Nodes.
      • etcd: Distributed key-value store for configuration data.

  4. Deployments

    • Define the desired state for Pods, such as which images to use and how many replicas.
    • Automatically manages updates and scaling.

  5. Services

    • Expose a set of Pods as a network service.
    • Types:
      • ClusterIP: Internal access within the cluster.
      • NodePort: Exposes a port on each Node.
      • LoadBalancer: Exposes the service via a cloud provider’s load balancer.

  6. Volumes

    • Persistent storage that Pods can access.
    • Types:
      • EmptyDir: Temporary storage.
      • PersistentVolume (PV): Managed storage resource.
      • PersistentVolumeClaim (PVC): Request for storage.

  7. ConfigMaps and Secrets

    • ConfigMaps: Store configuration data.
    • Secrets: Store sensitive data (e.g., passwords, API keys).

  8. Namespaces

    • Logical partitions within a cluster to organize resources.

  9. Horizontal Pod Autoscaling (HPA)

    • Automatically scales Pods based on resource utilization (e.g., CPU or memory).

  10. Ingress

    • Manages external HTTP/HTTPS access to services within a cluster, often with routing rules.

  11. StatefulSets

    • Manages stateful applications with persistent storage, unique network identities, and stable storage.
